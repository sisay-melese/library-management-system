# Library Management System

## Git Commands Used

### git init
Initialized a local Git repository to track project changes.

### git add
Staged files to prepare them for committing.

### git commit
Saved project snapshots with descriptive messages.

### git push
Uploaded local commits to the remote GitHub repository.

### git pull
Fetched and merged updates from the remote repository.

### git branch
Created and managed branches for feature development.

## Branching and Merging Flow

1. The main branch contains stable code.
2. A separate branch (LoanManagement) was created to implement loan-related features.
3. Changes were committed and pushed to GitHub.
4. A Pull Request was created to merge LoanManagement into main.
5. After approval, the PR was merged.
6. Latest changes were pulled back into the local main branch.
